# 生电：物流、分类与存储

标签：生电、物流、物品分类、漏斗、投掷器、水道、仓库、比较器

## 物流系统的分层

稳定仓库通常按四层设计：输入层、运输层、筛选层和存储层。不要把所有功能塞进一条漏斗线，否则任何一个满箱或错误物品都可能反向堵塞整个系统。

```text
输入箱 / 投掷器
    -> 水流、冰道、漏斗矿车或漏斗线
    -> 单物品过滤器 / 溢流保护
    -> 分类箱、缓存箱、堆叠箱
    -> 无法识别物品的溢出箱
```

## 常见单物品过滤器

传统分类器常在过滤漏斗中放置目标物品和占位物品，例如常见的 `41-1-1-1-1` 布局。比较器检测漏斗库存变化，达到阈值后短暂解锁下方漏斗，放行目标物品。

```text
if filter_hopper.signal_strength >= unlock_threshold:
    unlock_output_hopper_for_short_pulse()
else:
    keep_output_hopper_locked()
```

占位物品必须使用不会进入系统的命名物品或不可混叠物品，避免其他物品替换占位物。过滤器适用于可堆叠物品；不可堆叠物品、不同 NBT/组件物品、可变堆叠上限物品和版本变更后的物品数据需要专门设计。

## 溢流保护

没有溢流保护的分类器可能在上游持续投放错误物品时被填满，导致红石信号超过安全阈值并串线。工程上应保留：

- 分类器前的缓存容器。
- 最后一格的溢出箱或销毁/回收路线。
- 高价值物品的独立线路，不与普通堆叠物混用。
- 维护口，用于查看漏斗库存和红石状态。

## 运输方式选择

| 方式 | 优点 | 限制 | 适用场景 |
| --- | --- | --- | --- |
| 漏斗线 | 结构直观、可直接接容器 | 每格漏斗都会带来性能开销 | 短距离、低吞吐 |
| 水流/冰道 | 低成本、长距离效率高 | 需处理物品合并、分流和终点收集 | 大型仓库主干线 |
| 投掷器链 | 可控、可向上运输 | 需要脉冲与防堵逻辑 | 垂直运输、定量发射 |
| 漏斗矿车 | 抽取范围广 | 实体与轨道维护成本高 | 从多格容器或方块下方抽取 |

## 性能原则

漏斗会持续检查库存，是大型生电仓库的常见性能来源。可以用箱子或实体方块阻断不需要工作的漏斗、使用水流承担长距离运输、将高频物流拆分为多条支线，并避免无意义的循环投掷器。性能优化前先用开关或分区定位瓶颈，不要盲目拆除整套仓库。
