# Minecraft 基础与版本边界

标签：Minecraft、Java 版、基岩版、生存模式、创造模式、难度、时间

## 游戏的核心循环

Minecraft 是以方块世界为基础的生存与创造游戏。典型生存循环是：收集基础资源、制作工具、获得食物与庇护、探索洞穴和结构、升级装备、进入下界获取关键材料、寻找要塞并进入末地、挑战末影龙。创造模式则主要用于建造、测试红石和制作地图。

## 常见游戏模式

- 生存模式：需要管理生命、饥饿、装备和资源；大多数方块需要正确工具才能高效采集。
- 创造模式：可飞行并可直接获取方块，适合建筑和电路测试。
- 冒险模式：通常用于地图，破坏或放置方块受到物品标签限制。
- 旁观模式：可穿过方块观察世界，通常不能与方块交互。

## 难度与死亡

难度影响敌对生物生成、伤害、饥饿和部分机制。和平难度通常不会保留敌对生物；简单、普通、困难逐步提高生存压力。死亡后物品通常会掉落在死亡位置，除非启用了 `keepInventory` 游戏规则。物品区块未加载时通常不会继续消失；区块加载后才会开始计时。

## 时间、区块和刻

一个完整昼夜约为 24000 游戏刻。游戏通常以每秒 20 游戏刻运行，但服务器卡顿时真实时间会拉长。红石常用“红石刻”表示 2 个游戏刻；中继器最短延迟为 1 个红石刻。只有已加载区块会持续进行作物生长、红石更新和大多数生物行为。

## Java 版与基岩版

两者共享多数基础配方、维度和生物，但以下内容经常不同：红石更新顺序、刷怪规则、部分指令参数、村民机制、旁观模式支持情况、投掷物行为和服务器插件生态。回答版本敏感问题时，应先确认用户使用的是 Java 版还是基岩版，并询问大致版本号。

## 提问时的有效信息

为了得到准确建议，建议同时说明：版本与平台、单人或服务器、是否开启模组/数据包、目标是生存实用还是红石技术玩法、以及已有材料或错误现象。
