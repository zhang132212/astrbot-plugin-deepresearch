# 生电：红石时序、元件与算法

标签：生电、红石刻、比较器、中继器、漏斗时钟、观察者、活塞、Java 版、基岩版

## 计时单位

生电装置必须区分真实时间、游戏刻和红石刻：

```text
1 秒（理想 TPS） = 20 游戏刻
1 红石刻 = 2 游戏刻 = 0.1 秒
```

中继器每一档增加 1 个红石刻延迟，四档分别约为 0.1、0.2、0.3、0.4 秒。比较器的输出延迟为 1 个红石刻。服务器 TPS 低于 20 时，游戏刻仍按逻辑顺序执行，但真实时间会变慢；因此农场计时应优先以游戏刻而不是墙上时间理解。

## 比较器的精确逻辑

比较器读取后方主输入 `R`，以及左右两侧中较高的输入 `S`。信号范围均为 0 到 15。

```text
side_signal = max(left_signal, right_signal)

if mode == COMPARE:
    output = rear_signal if rear_signal >= side_signal else 0

if mode == SUBTRACT:
    output = max(rear_signal - side_signal, 0)
```

比较模式适合阈值门、锁存条件和信号选择；减法模式适合容器读数、计数器和物品过滤器。比较器读取容器时得到的是容器状态换算出的强度，不是物品 ID。

## 漏斗传输与漏斗时钟

标准漏斗有物品传输冷却。以常见 Java 版机制为例，一个漏斗在可传输时通常每 8 游戏刻尝试转移一个物品。因此双漏斗时钟中，若一侧有 `N` 个可移动物品：

```text
单程时间 ≈ 8 * N 游戏刻
完整往返周期 ≈ 16 * N 游戏刻
理想秒数 ≈ 0.8 * N 秒
```

伪代码可表示为：

```text
while clock_is_enabled:
    every 8 game_ticks:
        move_one_item(active_hopper, passive_hopper)
    if active_hopper.item_count == 0:
        swap(active_hopper, passive_hopper)
```

实际周期会受漏斗锁定、容器满载、区块卸载和服务器卡顿影响。用于关键定时器时，应在目标服务器环境实测。

## 观察者、活塞和脉冲

观察者在检测前方方块状态变化时从背面输出短脉冲，常用于甘蔗、竹子、熔炉状态、活塞移动和飞行器触发。观察者适合“状态变化”而不是“持续状态”；若要把短脉冲转换为稳定开关，需要使用锁存器或触发器。

活塞系统应分开理解三个状态：检测条件、延迟链、推动结果。推荐先把电路写成状态表：

| 当前状态 | 触发条件 | 输出动作 | 下一状态 |
| --- | --- | --- | --- |
| 待机 | 观察者脉冲 | 打开收集机构 | 收集 |
| 收集 | 延迟结束 | 关闭收集机构 | 待机 |

Java 版存在准连接等版本特性，基岩版的活塞更新与红石行为不同。跨版本复刻装置时，先确认设计明确标注了适用版本。

## 红石信号排错算法

```text
1. 测量每个关键点的信号强度，而非只看红石粉亮灭。
2. 检查中继器方向、档位和侧锁。
3. 检查比较器模式：前端火把亮起时为减法模式。
4. 检查是否存在短脉冲被延迟链吞掉或被锁存器过滤。
5. 检查装置所在区块、实体与方块是否仍在模拟范围内。
6. 最后再检查版本差异、服务器插件和 TPS。
```
