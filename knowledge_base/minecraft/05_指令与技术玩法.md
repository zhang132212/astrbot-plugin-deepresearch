# Minecraft 指令与技术玩法

标签：指令、命令方块、gamerule、locate、teleport、服务器、排错

## 使用前提

指令通常需要开启作弊、拥有管理员权限或在命令方块中执行。Java 版和基岩版的选择器、物品命名空间、坐标格式和部分子命令存在差异。给出可执行命令前，应明确版本和执行环境。

## 常见概念

- `@s`：命令执行者。
- `@p`：最近玩家。
- `@a`：所有玩家。
- `@e`：所有实体，可通过类型、距离、标签等条件筛选，具体语法依版本而异。
- 相对坐标使用 `~`，局部坐标使用 `^`；局部坐标尤其依赖执行者朝向。

## 常用示例

```text
# 保留死亡物品，Java/基岩版通常都支持此规则
/gamerule keepInventory true

# 传送自己到指定坐标
/tp @s 100 64 -200

# Java 版中给予钻石的常见写法
/give @s minecraft:diamond 1
```

`/locate`、`/execute`、`/data`、`/scoreboard` 和 `/function` 的细节版本差异较大。复杂命令应先在创造模式测试世界运行，再部署到服务器或地图。

## 命令方块与函数

命令方块可设置为脉冲、循环或连锁类型，并可配置是否需要红石。数据包函数适合把长命令拆分为可维护文件；基岩版则使用自身的行为包与函数体系。不要在高频循环命令中扫描大量实体，否则会造成明显性能问题。

## 技术玩法排错

当装置在单人有效、服务器失效时，优先检查：服务端版本、模拟距离、加载区块、权限插件、性能优化插件、游戏规则和红石实现差异。对重要世界进行命令、数据包或技术装置测试前，先备份存档。
